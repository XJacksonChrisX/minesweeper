### **Windows：安裝編譯器（第一次必做）**

如果看到「g++ not found」的錯誤，需要先安裝 MinGW-w64。

**最快方式（推薦）：用 Chocolatey**

1. **以管理員身份打開 PowerShell**

   - 右鍵選「以系統管理員身份執行」

2. **複製貼上這一行，按 Enter：**

   choco install mingw-w64 -y

3. **安裝完後，重開一個 CMD 視窗，重新執行 `build.bat`**

完成後，`build.bat` 就能正常執行了！

### **專案概述**
這是一個使用 C++ 開發的經典踩地雷遊戲。遊戲提供三個難度等級，玩家需要通過邏輯推理和運氣來揭露所有非地雷的格子，同時避免踩到地雷。

### **遊戲規則**
1. 玩家選擇難度等級後，遊戲會生成一個隨機布置地雷的棋盤
2. 玩家可以通過輸入座標來揭露格子
3. 揭露的格子會顯示：
   - **💣**：地雷（遊戲失敗）
   - **·**：空格（周圍沒有地雷）
   - **1-8**：相鄰地雷個數
4. 玩家也可以標記懷疑的地雷位置（**🚩**）
5. 揭露所有非地雷格子即獲勝

### **難度等級**
| 難度 | 棋盤大小 | 地雷數量 |
|------|---------|--------|
| 簡單 | 8×8 | 10 |
| 中等 | 12×12 | 30 |
| 困難 | 16×16 | 40 |

### **主要功能**

#### **1. Cell 類別**
表示棋盤上的單一格子，記錄：
- 是否為地雷
- 是否已被揭露
- 是否被標記
- 相鄰地雷數量

#### **2. Board 類別**
管理整個遊戲棋盤：
- 動態分配二維陣列存儲棋盤
- 隨機放置地雷
- 計算每個格子的相鄰地雷數
- 驗證座標有效性

#### **3. Game 類別**
核心遊戲邏輯：
- 根據難度初始化棋盤
- 處理玩家輸入命令
- 檢查遊戲勝敗條件
- 實現泛洪填充算法（自動揭露連通的空格）
- 與 Display 類別合作進行 UI 顯示

#### **4. Display 類別**
管理所有使用者介面與輸入輸出：
- 顯示棋盤（含列/行編號）
- 顯示遊戲歡迎訊息與操作菜單
- 顯示遊戲結果（勝利/失敗）
- 提示錯誤訊息
- 集中管理 UI 邏輯，易於擴展與維護

#### **5. main 函數**
遊戲入口點：
- 顯示難度選擇菜單
- 建立遊戲物件
- 啟動遊戲主迴圈

### **技術實現**

#### **使用的 C++ 特性**
- **動態記憶體管理**：使用指標管理棋盤二維陣列
- **類別與物件導向設計**：各功能模組化
- **標準模板庫 (STL)**：使用 `queue` 實現泛洪填充
- **結構化綁定**（C++17）：簡化資料結構操作

#### **核心算法**

**泛洪填充 (Flood Fill)**
當玩家揭露一個空格（周圍沒有地雷）時，自動揭露所有相鄰的格子，直到遇到有數字的邊界。這提高了遊戲體驗。

**相鄰地雷計算**
針對每個非地雷格子，檢查其 8 個相鄰方向，計算地雷個數。

## 程式設計概念應用

### **物件導向設計 (OOP)**
- **封裝 (Encapsulation)**：每個類別管理自己的數據和操作
- **繼承 (Inheritance)**：未來可擴展不同類型的遊戲模式
- **多型 (Polymorphism)**：可定義不同難度的初始化邏輯

### **資料結構**
- **二維陣列**：存儲棋盤狀態
- **佇列 (Queue)**：實現泛洪填充算法

### **演算法**
- **泛洪填充算法**：廣度優先搜索 (BFS)
- **隨機數生成**：地雷位置的隨機化
